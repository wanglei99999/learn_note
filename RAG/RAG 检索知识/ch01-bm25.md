要理解 BM25，先从它的前身 TF-IDF 说起。

### TF-IDF 的思路

信息检索的核心问题是：**给定一个查询，文档库里哪篇文档最相关？**

最朴素的想法：查询词在文档里出现越多，文档越相关。这就是 **TF（Term Frequency，词频）**：

$$TF(t, d) = \frac{词\ t\ 在文档\ d\ 中出现的次数}{文档\ d\ 的总词数}$$

但词频有个问题：停用词（"的"、"是"、"在"）天然高频，却毫无区分度。所以引入 **IDF（Inverse Document Frequency，逆文档频率）**：

$$IDF(t) = \log\frac{N}{df(t) + 1}$$

其中 N 是文档总数，df(t) 是包含词 t 的文档数。词越罕见，IDF 越高，权重越大。

$$TF\text{-}IDF(t, d) = TF(t, d) \times IDF(t)$$

**TF-IDF 的两个硬伤：**

1. **TF 无上限**：词出现 100 次的权重是出现 10 次的 10 倍——但实际上，重复出现的边际信息量在递减
2. **不考虑文档长度**：一篇 10000 词的文档里某词出现 5 次，和一篇 100 词的文档里出现 5 次，重要性完全不同

### 【必须掌握】BM25 如何解决这两个问题

**BM25（Best Match 25）** 的完整公式：

$$Score(D, Q) = \sum_{i=1}^{n} IDF(q_i) \cdot \frac{f(q_i, D) \cdot (k_1 + 1)}{f(q_i, D) + k_1 \cdot \left(1 - b + b \cdot \frac{|D|}{avgdl}\right)}$$

逐项拆解：

| 符号 | 含义 |
|------|------|
| $q_i$ | 查询中的第 i 个词 |
| $f(q_i, D)$ | 词 $q_i$ 在文档 D 中的原始词频 |
| $k_1$ | 词频饱和参数，默认 1.2，控制 TF 增长上限 |
| $b$ | 长度归一化强度，默认 0.75 |
| $\|D\|$ | 文档 D 的词数 |
| $avgdl$ | 语料库所有文档的平均词数 |

**k1 的作用——TF 饱和：**

$$饱和函数 = \frac{f \cdot (k_1 + 1)}{f + k_1}$$

当 f → ∞ 时，饱和函数 → (k₁ + 1)，即词频有了上限。k₁=1.2 意味着词频贡献最多增加到约 2.2 倍基础分，而不是无限增长。

**b 的作用——长度惩罚：**

分母中 $1 - b + b \cdot \frac{|D|}{avgdl}$：
- 当 b=0：完全不考虑文档长度
- 当 b=1：完全按文档长度归一化
- 当 b=0.75（默认）：部分归一化，长文档的词频会被适度打折

**IDF 的 BM25 版本**（与 TF-IDF 略有不同）：

$$IDF(q_i) = \ln\left(\frac{N - df(q_i) + 0.5}{df(q_i) + 0.5} + 1\right)$$

这个版本在极端情况（词出现在所有文档中）下不会产生负数。

### 另一个角度：把 BM25 看作对 IDF 的逐步"打补丁"

$$IDF \xrightarrow{+TF} \xrightarrow{+饱和} \xrightarrow{+长度归一化} BM25$$

| 步骤 | 解决的问题 | 引入的参数 |
|------|-----------|-----------|
| IDF | 常见词权重低，罕见词权重高 | — |
| + TF | IDF 不知道词在文档里出现了几次 | $f(q_i, D)$ |
| + TF 饱和 | 词频无上限，100 次 = 10 次 × 10，但实际边际递减 | $k_1$ |
| + 长度归一化 | 长文档词频天然偏高，不代表更相关 | $b$ |

这个**构造式视角**回答了"BM25 比 TF-IDF 多解决了什么问题"，直觉清晰，调参时很好用。

但它没有回答"饱和函数为什么偏偏是 $\frac{f(k_1+1)}{f+k_1}$ 这个具体形式，而不是 $\log(f+1)$ 或别的曲线"——这是下面 BIM 概率推导要回答的。

**🔬 研究员**

BM25 有完整的概率论根基。它来自 1970 年代在信息检索领域发展出来的**概率相关性框架（Probabilistic Relevance Framework）**，核心祖先是 **BIM（Binary Independence Model，二元独立模型）**。

理解这个推导过程，才能真正明白 BM25 的每个参数为什么要这样设计。

### 概率排名原则（Probability Ranking Principle）

Robertson 在 1977 年提出：**按照文档与查询相关的概率 P(R=1|d,q) 对文档排序，是最优的检索策略。**

这把检索问题从"计算相似度分数"变成了"估计条件概率"。现在的问题是：怎么计算 P(R=1|d,q)？

### 【理解即可】BIM：用贝叶斯公式建模

> **为什么要理解这个推导？** 它回答了"BM25 的 IDF 公式为什么长这样，TF 饱和函数为什么是 $\frac{f(k_1+1)}{f+k_1}$ 而不是 log(f)"。不需要记住推导细节，但要记住三个结论：**IDF 来自 RSJ 权重**（相关/不相关文档的对数比）、**k₁ 来自精英度假设**（词频边际递减）、**b 来自两种长度效应的权衡**。

BIM 由 Robertson & Sparck Jones 在 1976 年提出，做出三个假设：

1. **词袋假设**：文档用每个词是否出现（1/0）表示，忽略词序
2. **条件独立假设**：给定相关性 R，各词出现与否相互独立
3. **二元假设**：每个词只有"出现"和"不出现"两个状态

设：
- $p_i = P(t_i = 1 \mid R=1)$：词 $t_i$ 在相关文档中出现的概率
- $u_i = P(t_i = 1 \mid R=0)$：词 $t_i$ 在不相关文档中出现的概率

把文档 $d$ 写成一个二元向量 $x=(x_1,\dots,x_m)$，其中：

$$
x_i =
\begin{cases}
1, & t_i \in d \\
0, & t_i \notin d
\end{cases}
$$

对排序来说，我们不必直接求绝对概率 $P(R=1 \mid d,q)$，只需要比较相关/不相关的**赔率**：

$$
\frac{P(R=1 \mid d,q)}{P(R=0 \mid d,q)}
=
\frac{P(d \mid R=1,q)\,P(R=1 \mid q)}{P(d \mid R=0,q)\,P(R=0 \mid q)}
$$

在"给定相关性后各词条件独立"的假设下：

$$
P(d \mid R=1,q) = \prod_i p_i^{x_i}(1-p_i)^{1-x_i}
$$

$$
P(d \mid R=0,q) = \prod_i u_i^{x_i}(1-u_i)^{1-x_i}
$$

取对数后，乘积变成加法：

$$
\log \frac{P(R=1 \mid d,q)}{P(R=0 \mid d,q)}
=
\log \frac{P(R=1 \mid q)}{P(R=0 \mid q)}
+ \sum_i x_i \log \frac{p_i}{u_i}
+ \sum_i (1-x_i) \log \frac{1-p_i}{1-u_i}
$$

把与文档无关的项合并成常数 $C(q)$，就得到：

$$
\log \frac{P(R=1 \mid d,q)}{P(R=0 \mid d,q)}
=
C(q) + \sum_i x_i \log \frac{p_i(1-u_i)}{u_i(1-p_i)}
$$

对于同一个查询，$C(q)$ 对所有文档都一样，排序时可以直接丢掉。于是最终文档得分正比于：

$$Score_{BIM}(d, q) = \sum_{t_i \in q \cap d} \log \frac{p_i(1 - u_i)}{u_i(1 - p_i)}$$

这就是 **RSJ 权重（Robertson-Sparck Jones Weight）**：词的权重由它在相关文档 vs 不相关文档中出现频率的对数比决定。

### RSJ 权重 → BM25 的 IDF

如果把上面的单词权重单独拿出来写，就是：

$$
w_i = \log \frac{p_i(1-u_i)}{u_i(1-p_i)}
$$

关键问题来了：真实系统里，我们通常并不知道 $p_i$ 和 $u_i$。

#### 没有 relevance feedback 时，IDF 会自然长出来

在冷启动场景里，可以做两个经典近似：

1. 用语料统计估计"词在非相关文档中出现的概率"：$u_i \approx \frac{n_i}{N}$
   其中 $n_i = df(t_i)$，表示包含词 $t_i$ 的文档数，$N$ 是总文档数
2. 先忽略不同词之间 $p_i$ 的差异，把它近似看成一个常数

这样一来，和排序真正相关的部分就近似只剩：

$$
w_i \approx \log \frac{1-u_i}{u_i}
$$

把 $u_i = \frac{n_i}{N}$ 代进去：

$$
w_i \approx \log \frac{1-\frac{n_i}{N}}{\frac{n_i}{N}}
= \log \frac{N-n_i}{n_i}
$$

这就是最朴素的 IDF 形式：词越少见，权重越大；词越常见，权重越小。

#### 如果有相关性反馈，就得到更一般的 RSJ 估计

如果存在 relevance feedback，可以进一步用反馈数据估计 $p_i$ 和 $u_i$：

$$p_i \approx \frac{r + 0.5}{R + 1}, \quad u_i \approx \frac{df - r + 0.5}{N - R + 1}$$

其中 $r$ 是相关文档中包含词 $t_i$ 的数量，$R$ 是相关文档总数。

在没有任何相关性反馈时（$r=0, R=0$），它会退化成：

$$w_{RSJ}(t) \approx \log \frac{N - df(t) + 0.5}{df(t) + 0.5}$$

**这正是 BM25 的 IDF 公式来源。** 分子分母各加 0.5 通常叫 **0.5 correction**，本质上是一种贝叶斯平滑：防止极端情况（词从未出现、或出现在所有文档中）让对数比发散。

### BIM 的致命缺陷

BIM 只看词**是否出现**（0 或 1），完全忽略了词频。但直觉告诉我们：一篇文章里"机器学习"出现 10 次和出现 1 次，相关性是不一样的。

Robertson 等人在 1994 年的 Okapi 系统中引入了**精英度假设（Eliteness Assumption）**：

> 词频不直接决定相关性，但词频是"精英度"（即该词是否是文档主题词）的一个指示变量。高词频 → 更可能是主题词 → 更可能相关。

但这个关系是**边际递减的**：从出现 1 次到出现 2 次，信息量很大；从出现 99 次到 100 次，几乎没有新信息。

这就引出了 TF 饱和函数的设计需求。

### 从精英度假设推导 TF 饱和函数

在精英度模型中，引入隐变量 E（文档是否以该词为主题词），词频 f 通过 E 间接影响相关性：

$$P(R=1 \mid f) \propto \frac{f \cdot c_1}{f \cdot c_1 + (1-E) \cdot c_2}$$

这个函数的形状，在数学上正比于一个有上限的饱和曲线。Robertson 发现，一个简洁的近似是：

$$\text{TF saturation} = \frac{f \cdot (k_1 + 1)}{f + k_1}$$

当 f → ∞，饱和值趋近 $(k_1 + 1)$，即词频的贡献有上限。

这不是拍脑袋的公式，而是对精英度模型的**数学近似**。

### 长度归一化：为什么存在 b

BIM 也没有考虑文档长度。长文档词频天然偏高——10000 词的文档出现"合同"5 次，可能不如 500 词文档出现"合同"3 次更专注。

但长文档还有一个合理面：它覆盖的内容就是更多。Robertson 认为有两种原因导致长文档词频高：

1. **范围效应（Scope）**：文档讨论了更多主题，词只是"顺带"出现
2. **详细程度效应（Verbosity）**：讨论同一主题时，表达更啰嗦

前者应该归一化（不该因为多主题就加权），后者不该归一化（表达详细是内容丰富）。

参数 `b` 就是在这两种效应之间**线性插值**：

$$\text{归一化因子} = (1 - b) + b \cdot \frac{|D|}{avgdl}$$

- b=0：不归一化（假设文档长度全是详细程度效应）
- b=1：完全归一化（假设文档长度全是范围效应）
- b=0.75（默认）：大部分情况长文档的长是范围效应，少量是详细程度

### BM25 命名的由来

BM25 中的 "25" 是字面意思——这是 1980~1994 年间 Okapi 项目尝试的第 25 个"最佳匹配（Best Match）"公式变体。早期版本包括：

| 版本 | 特点 |
|------|------|
| BM11 | b=1（完全按文档长度归一化）|
| BM15 | b=0（完全不归一化）|
| **BM25** | 0<b<1（平衡两种效应，最终胜出）|

BM25 由 Robertson, Walker, Jones, Hancock-Beaulieu, Gatford 在 1994 年 TREC-3 上正式发表，此后成为文本检索的黄金标准，延续至今。

**❓ 提问者**

所以 BM25 IDF 里分子分母各加 0.5，不是随意的，而是贝叶斯估计里的平滑项？

---

**🔬 研究员**

完全正确。加 0.5 是**贝叶斯先验带来的平滑修正**，不是随手拍进去的经验常数。

更精确地说：这相当于在 $p_i$ 和 $u_i$ 的最大似然估计上加一个对称的 Beta(0.5, 0.5) 先验（Jeffreys 先验）。它和最常见的 `+1` 拉普拉斯平滑不是一回事，但作用类似：在极端频率下防止概率变成 0 或 1，使对数比不发散。

在实际工程中，当 df(t) = N（词出现在所有文档）时：

```python
# 不加平滑
IDF = log((N - N) / N) = log(0) = -∞  # 炸了

# 加 0.5 平滑
IDF = log((N - N + 0.5) / (N + 0.5)) ≈ log(0.5 / N) < 0
# 仍然是小的负数，但不会崩溃
```

Lucene 的实现为了简洁，在外面加了 +1 保证 IDF 恒正：

$$IDF_{Lucene}(t) = \ln\left(1 + \frac{N - df(t) + 0.5}{df(t) + 0.5}\right)$$

这样即使词出现在所有文档，IDF 也是 ln(1) = 0 而不是负数，避免了该词拉低总分。ES 默认就是这个版本。

### 理论对工程的实际指导

理解了这个推导之后，调参就不再是玄学：

**k1 控制"精英度敏感性"**：k1 越大，词频区分度越高（到达饱和越慢）；k1 越小，几乎只看词是否出现（退化向 BIM）。

```
k1 = 0:  f/(0+f) = 1，所有词频都变成 1，等同 BIM
k1 = ∞:  f·∞/(f+∞) ≈ f，退化为线性 TF，无饱和
k1 = 1.2: 经验最优，词频到 3~5 次基本饱和
```

**b 控制"假设文档长度的成因"**：

```
b → 0:  相信"长文档是因为表达详细，每个词都更有信息量"
b → 1:  相信"长文档是因为话题宽泛，词的出现稀释了"
b = 0.75: 实验证明对大多数语料库最优的折中
```

| 语料特点 | 推荐 b | 理由 |
|---------|--------|------|
| 法律条文（相同主题，表达详细） | 0.3~0.4 | 长度更多是 Verbosity |
| 新闻摘要（短，话题集中） | 0.7~0.9 | 长度更多是 Scope |
| 技术文档（混合） | 0.5~0.75 | 折中 |

### 中文分词是 BM25 的命脉

BM25 统计的是 token 的频率，分词粒度直接决定效果。

**默认 standard 分析器**对中文是按单字切分：
```
"深圳市财政局" → ["深", "圳", "市", "财", "政", "局"]
```
这会导致"深圳"和"深度"的"深"字被当成同一个 token 匹配，召回大量噪声。

**正确做法：安装 IK 分词器**

```bash
# ES 容器内安装
./bin/elasticsearch-plugin install \
  https://github.com/medcl/elasticsearch-analysis-ik/releases/download/v8.x.x/elasticsearch-analysis-ik-8.x.x.zip
```

IK 有两种模式：
- `ik_max_word`：最细粒度切分（入库时用，保证召回）
- `ik_smart`：最粗粒度切分（查询时用，保证精度）

```json
PUT /arc_chunks
{
  "settings": {
    "analysis": {
      "analyzer": {
        "ik_index": { "tokenizer": "ik_max_word" },
        "ik_search": { "tokenizer": "ik_smart" }
      }
    }
  },
  "mappings": {
    "properties": {
      "content": {
        "type": "text",
        "analyzer": "ik_index",
        "search_analyzer": "ik_search"
      }
    }
  }
}
```

入库用最细粒度，查询用最粗粒度——这是标准的"大索引小查询"策略。

**🔬 研究员**

完全正确，这是 BM25 的 **词汇鸿沟（Vocabulary Mismatch）** 问题，属于结构性缺陷。

有两种传统的局部缓解方案：

1. **同义词词典**：在 ES 分析器中配置同义词
   ```json
   "filter": {
     "synonym_filter": {
       "type": "synonym",
       "synonyms": ["取消订单, 退款, 退货"]
     }
   }
   ```
   缺点：需要人工维护，领域变化后要持续更新。

2. **查询扩展**：用 WordNet 或领域词典自动扩展查询词。
   缺点：可能引入噪声，"苹果"扩展为"水果"会带来无关结果。

但这两种方案治标不治本，真正的解决方案是**向量检索**——语义在向量空间里天然对齐。这也是为什么现代 RAG 必须混合两者。
